# 安全漏洞报告

Arcentra 团队与社区非常重视安全漏洞。感谢你以负责任的方式披露问题，我们会尽最大努力及时确认、跟进并修复。

## 如何报告漏洞

推荐方式：使用 GitHub Security Advisories。

- 进入 GitHub 仓库的 **Security** 页签，点击 **Report a vulnerability**。
- 若可用，可直接访问：`https://github.com/arcentrix/arcentra/security/advisories/new`

备用方式：如果你无法使用 GitHub Security Advisories，也可以发送邮件到 `support@arcentra.io` 进行报告。

建议在报告中包含：

- 漏洞描述与影响范围
- 受影响版本（如已知）
- 复现步骤 / PoC（如可提供）
- 可能的修复建议或缓解措施

## 我们的响应流程

- 我们会尽快确认收到报告，并告知后续处理步骤。
- 在首次回复之后，我们会持续同步修复进展，并协调漏洞披露节奏。
- 视情况可能会向你请求更多信息以便验证与修复。

## 第三方依赖

若漏洞来自第三方依赖，建议同时向上游维护者报告。如不确定归属，也欢迎先报告给 Arcentra，我们会协助分流与定位。
